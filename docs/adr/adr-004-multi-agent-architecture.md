# ADR-004: 多智能体架构而非单体 LLM 调用

## 状态

✅ 已采纳 (2026-05-29)

## 背景

面试系统需要完成出题、评分、报告等多个任务，每个任务需要的 prompt 和逻辑不同。

## 方案对比

| 维度 | 多智能体 | 单体 Prompt | 流水线函数 |
|------|---------|-------------|-----------|
| 职责分离 | 清晰 | 混杂 | 中等 |
| 可测试性 | 独立测试 | 集成测试 | 函数级测试 |
| 可扩展性 | 加 Agent 即可 | 修改巨大 prompt | 改中间步骤 |
| 复杂度 | 高 | 低 | 中 |

## 决策

选 **多智能体**。

## 理由

1. **单一职责** — 每个 Agent 只做一件事（出题、评分、分析简历、写报告）
2. **独立开发** — 4 个 Agent 可以并行开发、独立测试
3. **Prompt 隔离** — 出题 prompt 出错不影响评分 prompt
4. **SharedMemory 模式** — Agent 通过共享内存通信，Orchestrator 不需要传参数

## 后果

- 架构复杂度较高，新开发者需要理解 Agent 通信机制
- Orchestrator 需要管理 Agent 生命周期
- Agent 间通信通过 SharedMemory + MessageBus，调试时需要跟踪事件流
